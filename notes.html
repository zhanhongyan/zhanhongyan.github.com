2015-04-20My First GitHub Website

Git Repository : 仓库；
Commit to master : 提交修改文件至（自己创建的）master；
Create Pull Request : 向（项目组master）提交自己Fork Branch的修改，master创建人会对该次请求进行审核；
Git Push : 将本地分支的更新推送到远程主机；
Git Fetch : 将更新后的版本从远程主机下载到本地；
Git Reset : 将当前目录完全回滚到reset指定的版本号，其后的commit都将被放弃；
Git Revert : 只撤销当前commit，不影响之前的commit；
Git Sync : 将本地修改同步至远程服务器；

2015-04-16H5页面设计心得

想法  被大众主流乐于分享的信息大概有以下几种：
1、最直接的利益驱动，分享有奖或者需要众筹等。
2、对自己有用并且对周围人也会有新鲜感的信息。
3、自己参与游戏或者测试得到的结果。
跳出来去看你做的东西，首先一点是你和你周围的人愿意转发分享这个东西吗?它有趣吗?它对其他人也是有用的信息吗?

 

版式  在有限的手机屏幕空间内，最好的版式风格是简约。
1、最好有一个核心元素，中心元素不宜过多，突出重点为优。
2、简约不是简单的删减，而是提炼内容精华以作展示。简约设计具有清晰的页面结构和简单的交互操作。
3、简约风格的主要表现方式：大标题，大背景，留白，单纯的色彩，轻薄的按钮……

2015-04-15JavaScript自学笔记一

定义  JavaScript于1995年由Netscape公司设计而成，兼容于ECMA（欧洲计算机制造商协会）标准。
1、JavaScript是一种直译式脚本语言（代码不执行预编译），浏览器会在读取代码时逐行地执行脚本代码。如果一条运行不了，那么下面的代码也无法运行。
2、主要用来向HTML（超文本标记语言）页面添加交互行为，对浏览器事件做出响应，控制浏览器cookies，基于node.js技术进行服务器端编程。
3、可以直接嵌入HTML页面，但写成单独的JS文件有利于结构和样式的分离。
4、JavaScript作为客户端脚本语言在用户的浏览器运行，不依赖于操作系统。

 

语法  JavaScript是一种基于对象的脚本语言,它不仅可以创建对象,也能使用现有的对象。
1、JavaScript 语句通过代码块的形式进行组合。块由左花括号开始，由右花括号结束。
2、JavaScript语句对大小写敏感，函数 getElementById 与 getElementbyID 是不同的。
3、JavaScript会忽略多余的空格，可以在脚本中添加空格以提高可读性。
4、使用document.write()向文档输出内容，如果在文档已完成加载后执行document.write()，整个HTML页面将被覆盖。

 

变量  在 JavaScript 中创建变量通常称为“声明”变量。我们使用 var （variable）关键词来声明变量。
1、在代码开始处，统一对需要的变量进行声明
2、JavaScript变量对大小写敏感。所有的变量在定义时都要加上var。
        var x   // x 为 undefined
        var x=2;  // x 为数字
        var name="Jason";  // x 为字符串
3、JavaScript 变量均为对象。当您声明一个变量时，就创建了一个新的对象。

2015-04-08常用平面设计物料尺寸及印刷工艺

名片  AI源文件出血2mm，300dpi
中式直角名片 成品尺寸：90mm×54mm，是最常见的名片尺寸。
美式直角名片 成品尺寸：90mm×50mm，略窄，适合女士。
欧式圆角名片 成品尺寸：86mm×54mm，常用于公交卡、银行卡。
普通名片选用300g铜版纸，不覆膜或覆哑膜（亮膜慎用，成品塑料感太强）。要求高者可选特种纸，常用工艺有覆膜、过光油、烫金烫银、凹凸、折叠、镂空、异形切等。

 

画册  AI源文件出血3mm，300dpi
最常用尺寸为16开画册（210mm×285mm），P数为4的倍数，页数较少时用骑马钉，页数较多时需使用胶装或锁线胶装。
画册内页最常用157g铜版纸或哑粉纸，也可选择128g或200g纸；封面封底用纸通常比内页稍厚，可选200g~300g等纸张。封面封底过光胶（覆膜），内页过油，会使整本画册看上去更加绚丽。

2015-03-31HTML中<b>标签与<strong>标签的区别

1、<b>标签是html的内容，而<strong>标签是XHTML的内容，从标准化上来看，<strong>标签更胜一筹。

2、<b>是一个功能标签，仅在视觉效果上加粗标签内文字，本身不具备HTML语义。而<strong>是一个逻辑标签，表示语气上的强调、加重，表明标签内文字在上下文环境中的重要性。从搜索引擎的人性化上，<strong>标签更加适用。

3、WEB标准提倡结构、样式和行为的分离，所以纯粹为了视觉上加粗文字而使用<b>标签是非常不划算的。

4、当盲人适用屏幕阅读器时，遇到<b>标签会像处理一般词语一样阅读，而遇到<strong>标签会加重与停顿。

总结：从WEB标准和用户体验性上来说，<strong>标签更有益，更被推荐使用。

2015-03-31子DIV有浮动时无法撑开父DIV的解决办法

      1. float;
      缺点：层层往上找，没完没了，到了body那，算是个头 float 和 margin:0 auto; 有冲突
      2. display:inline-block;
      缺点：不能设置 margin:0 auto;
      3. 父DIV添加overflow:hidden;
      缺点：很多JS交互特效做不了，多出去的部分隐藏了
      4. 在父级内，加空DIV：clear:both;
      缺点：父级不能加padding，否则IE6 IE7不兼容
      5. 在父DIV的闭合标签前<br clear="all" />
      缺点：每次都要加这个换行符
      6. .clear:after { content: ''; clear: both; display: block; }
      此外，如果父级没有宽，就必须为父级加上：zoom:1;用来解决IE6 IE7下的问题
